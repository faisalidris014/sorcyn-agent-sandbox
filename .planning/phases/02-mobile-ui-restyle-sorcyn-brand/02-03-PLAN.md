<objective>
Wire the spring page transition route builder (created in Plan 01) into every standalone
GoRoute in `app.dart`. Without this, the locked Sorcyn animation token "Spring page
transitions (stiffness 320, damping 32)" from ROADMAP Phase 2 SC #4 is unreachable.

Before:
```dart
GoRoute(path: '/posts/:postId', builder: (context, state) => PostDetailScreen(...))
```

After:
```dart
GoRoute(
  path: '/posts/:postId',
  pageBuilder: (context, state) => springPage(
    key: state.pageKey,
    child: PostDetailScreen(postId: state.pathParameters['postId']!),
  ),
)
```

Routes affected (all standalone routes, NOT the StatefulShellBranches):
- `/login`, `/register`, `/auth/verify-email`, `/auth/forgot-password`, `/auth/reset-password`
- `/posts/create`, `/posts/create/ai`, `/posts/create/manual`
- `/posts/created/:postId`, `/posts/:postId/edit`, `/posts/:postId`
- `/posts/:postId/offers`, `/posts/:postId/submit-offer`, `/offers/:offerId`
- `/transactions`, `/transactions/:transactionId`, `/transactions/:transactionId/seller`
- `/transactions/:transactionId/dispute`, `/transactions/:transactionId/review`
- `/chat/:conversationId`
- `/seller/profile/setup`, `/seller/profile`, `/seller/verification`, `/seller/stripe-onboard`
- `/notifications`, `/profile/edit`
- `/settings`, `/settings/language`
- `/users/:userId`, `/payment-methods`, `/help`, `/seller/earnings`

The 4 StatefulShellBranches (`/dashboard`, `/my-posts`, `/messages`, `/profile`) MUST
continue to use `builder:` (not `pageBuilder:`) — the StatefulShellRoute uses its own
IndexedStack-based switcher and does not support per-route page transitions; switching
tabs in BuyerDashboard.tsx is also instant, so this is design-correct.

Purpose: Close ROADMAP SC #4 ("spring page transitions ... applied via a shared route builder").
Output: `app.dart` with `pageBuilder + springPage` on all 32 standalone routes.
</objective>

From mobile/lib/shared/transitions/spring_page_transition.dart (created in Plan 01):
```dart
CustomTransitionPage<T> springPage<T>({
  required LocalKey key,
  required Widget child,
  Duration? duration, // optional override; default _kTransitionDuration
});
```

Usage pattern in GoRouter (see go_router 14.x docs):
```dart
GoRoute(
  path: '/example',
  pageBuilder: (context, state) => springPage(
    key: state.pageKey,
    child: const ExampleScreen(),
  ),
),
```

`state.pageKey` is the canonical LocalKey GoRouter generates per route — passing it
ensures Hero animations and lifecycle hooks behave correctly.

Existing app.dart structure (relevant excerpt):
