<objective>
Implement the **positive half** of REQ-three-day-exclusivity: the targeted-seller carve-out (Addendum A-03 — category match definition resolves Open Question #1). The exclusion half (post-3-day filter) is already shipped at `posts.service.ts:516-520`; this plan adds the inclusion half so sellers whose `categories` overlap a post's `categoryId` see that post during the buyer's 3-day exclusivity window.

Per Addendum A-01 (PostGIS deferred to Phase 6), the shipped Haversine SQL radius filter and shipped `backend/src/config/geocoding.ts` utility are LEFT UNTOUCHED. This plan does NOT migrate to PostGIS, does NOT rewrite the radius filter, does NOT change geocoding. Verification of the shipped surfaces lands in Plan 08 audit assertions.

The seller feed route (`GET /feed`) is currently unauthenticated. To enable the targeted-seller carve-out without breaking unauthenticated callers, the route accepts **optional authentication**: if a Bearer token is present, it is parsed via `request.jwtVerify()` in a try/catch; if absent or invalid, the request proceeds without `requestingUserId` and the existing exclusion-only behavior applies.

Purpose: Close the inclusion-half gap so curated category-matched sellers see in-window posts (Phase 3 Success Criterion 1 + 3).
Output: Service-layer `where.OR` extension in getFeed + optional auth wrapper at the feed route + carve-out integration test.
</objective>
