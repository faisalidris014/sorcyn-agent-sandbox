<objective>
Implement REQ-pii-blocked-until-payment per addendum-confirmed decisions D-11, D-12, D-13. Pre-payment, sellers see only city + state + zip + distance from a buyer's post; the full street address is hidden until escrow is funded (transaction.escrowStatus in ['held', 'released']).

Per D-12, redaction lives at the **service layer** (not in DB views) so there's a single source of truth. The pattern mirrors the shipped `stripBudgetForNonOwner()` helper in posts.service.ts.

Per D-13, the mobile post detail screen surfaces a lock icon + "Full address shared after payment" microcopy when the address is null — so sellers don't think the missing field is a data bug.

Note: `locationCity`, `locationState`, `locationZip` are ALREADY exposed in `toPostResponse()` (lines 835-836 per PATTERNS.md) — those fields stay visible. Only the `locationAddress` (street address) is redacted. Buyer phone/email are not currently joined into post or offer responses — verify during Task 1; if they leak elsewhere, redact at the same callsite.
